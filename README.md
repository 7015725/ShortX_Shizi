# 识字_文字选择工具

ShortX / Rhino ES5 悬浮文字选择工具。用于从剪贴板文本中逐字选择、复制，并支持翻译后替换原选区。

来源标注：阿然博客 xin-blog.com  
项目说明：本仓库版本由原「拾字-百度翻译版.js」「拾字-有道翻译版.js」合并整理而来。

## 文件说明

- `识字_文字选择工具.js`：ShortX 脚本主文件，也是仓库唯一脚本文件。

## 功能概览

### 双翻译引擎

- 支持百度翻译与有道翻译。
- 配置项 `TRANSLATE_API`：
  - `1` = 百度翻译
  - `2` = 有道翻译
- 支持通过 ShortX 局部变量覆盖配置：
  - `localVarOf$翻译引擎`
  - `localVarOf$应用ID`
  - `localVarOf$应用秘钥`
- 百度使用 MD5 签名，有道使用 SHA-256 v3 签名。

### DIY 配置区

脚本顶部提供 `DIY_CONFIG`，便于调整常用参数：

- `MAX_CHAR_LIMIT`：最大载入字符数，当前为 `8000`。
- `TEXT_AREA_HEIGHT_DP`：文本展示区高度，当前为 `420dp`。
- `SCROLL_DELAY_MS`：边缘自动滚动刷新延迟，当前为 `10ms`。

### 文字选择与预览

- 支持长按拖选文字。
- 支持放大镜圆形预览。
- 支持选区预览、复制、清空、全选。
- 支持顶部字号设置，并保存字号偏好。

### 长文本加载

- 长文本首屏先显示前段内容。
- 主界面显示后自动补全完整文本。
- 拖选中或已有选区时，补全会延后，避免打断当前选择。

### 翻译替换与撤销

- 翻译结果可直接替换当前选区。
- 不连续选区会按连续 range 分段翻译、分段替换，不会把未选中文本送去翻译。
- 多段翻译串行执行，段间自动限速；遇到接口频率限制会自动退避重试。
- 整行选择但未选中行尾换行时，会保留原有换行结构，避免译文和下一行粘连。
- 替换后按钮切换为 `↶ 撤销`，可恢复替换前文本、选区、滚动位置与长文本加载状态。
- 内置并发请求保护，避免重复触发翻译。

## ShortX 变量配置建议

在 ShortX 规则/动作中准备以下局部变量：

- `剪贴板`：传入待处理文本；为空时脚本会尝试读取系统剪贴板。
- `翻译引擎`：`1` 百度，`2` 有道。
- `应用ID`：百度 AppID 或有道 AppKey。
- `应用秘钥`：对应平台的密钥。

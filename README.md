# 识字_文字选择工具

ShortX / Rhino ES5 悬浮文字选择工具。用于从剪贴板文本中逐字选择、复制，并支持翻译后替换原选区。

来源标注：阿然博客 xin-blog.com  
项目说明：本仓库版本由原「拾字-百度翻译版.js」「拾字-有道翻译版.js」合并升级而来。

## 文件说明

- `识字_文字选择工具.txt`：用户上传的原始文件，适合直接导入/复制到 ShortX。
- `识字_文字选择工具.js`：同内容 JS 后缀副本，便于 GitHub 代码高亮与版本管理。

## 本次更新内容

### 1. 双翻译引擎合并

- 将原来的百度版、有道版合并为一个脚本。
- 新增配置项 `TRANSLATE_API`：
  - `1` = 百度翻译
  - `2` = 有道翻译
- 统一读取 ShortX 局部变量：
  - `localVarOf$翻译引擎`
  - `localVarOf$应用ID`
  - `localVarOf$应用秘钥`
- 保留百度 MD5 签名逻辑，并新增/整合有道 SHA-256 v3 签名逻辑。

### 2. DIY 配置区增强

新增集中配置区 `DIY_CONFIG`，方便在文件顶部快速调整：

- `MAX_CHAR_LIMIT`：最大加载字符数，当前为 `8000`。
- `TEXT_AREA_HEIGHT_DP`：文本展示区高度，当前为 `420dp`。
- `SCROLL_DELAY_MS`：边缘自动滚动刷新延迟，当前为 `10ms`。

### 3. 放大镜选字预览

- 新增手指拖选放大镜预览层。
- 使用全屏透明悬浮容器 + 内部圆形 View 移动，减少高频 `WindowManager.updateViewLayout` 带来的卡顿。
- 放大镜内容使用镜像 TextView，同步选区高亮。
- 对内容重绑、Span 更新、位置移动都做了节流处理。

### 4. 拖选性能优化

- 增加 `DRAG_UPDATE_INTERVAL`、`pendingDragUpdate` 等拖选更新节流逻辑。
- 选区状态改用 `selectedSet` 管理，再转换为排序数组，降低频繁数组查找成本。
- 直接修改 TextView 当前 `Spannable` 的 Span，并通过 `invalidate()` 重绘，避免整段文本反复 `setText`。

### 5. 边缘自动滚动优化

- 增加上下边缘触发区：`SCROLL_EDGE_TOP` / `SCROLL_EDGE_BOTTOM`。
- 使用 `exactScrollY` 浮点累加滚动距离，让自动滚动更顺滑。
- 自动滚动时持续根据手指位置更新选区和放大镜。

### 6. 自适应 UI

- 新增屏幕尺寸检测：小屏、普通手机、大屏手机、平板、大平板。
- 根据设备类型动态调整 `uiScale`、窗口宽度、高度、字号、间距。
- 文本区高度根据内容和最大高度自动调整。

### 7. 字号设置持久化

- 字号支持通过顶部设置按钮实时调整。
- 优先保存到 `shortx.getShortXDir() + "/data/pickword_font_size.txt"`。
- 同时兼容 Android `SharedPreferences` 保存。

### 8. 翻译后替换与撤销

- 翻译结果会直接替换当前选区。
- 新增 `lastTranslationState` 保存替换前状态。
- 翻译后按钮变为 `↶ 撤销`，可恢复翻译前文本与选区。

### 9. 交互与容错优化

- 增加并发翻译锁 `isTranslating`，避免重复请求。
- 增强剪贴板读取、复制、悬浮窗权限检查。
- 创建/显示/更新放大镜时增加异常保护，避免单点失败导致脚本整体崩溃。
- 关闭窗口时清理放大镜、自动滚动、长按、保活回调等资源。

## ShortX 变量配置建议

在 ShortX 规则/动作中准备以下局部变量：

- `剪贴板`：传入待处理文本；为空时脚本会尝试读取系统剪贴板。
- `翻译引擎`：`1` 百度，`2` 有道。
- `应用ID`：百度 AppID 或有道 AppKey。
- `应用秘钥`：对应平台的密钥。
